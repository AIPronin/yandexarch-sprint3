# Задание 1. Анализ и планирование


### 1. Описание функциональности монолитного приложения

**Управление отоплением:**

- Пользователи могу управлять отоплением в доме через приложение. Пользователь может включать и отключать отопление в доме, а так же устанавливать нужную температуру и получать текущую установленную температуру системы отопления.
- Подключение к системе новых систем отопления происходит через вызов специалиста. Самостоятельно подключить свой датчик к системе пользователь не может.
- Система поддерживает получение данных от устройств при помощи синхронных вызовов от сервера к датчику
- Система поддерживает получение данных пользователем о состоянии системы отопления
- Система поддерживает возможность отправки команд пользователем по управлению системой отопления

**Мониторинг температуры:**

- Пользователи могу получить информацию о температуре в помещении через приложение. 
- Подключение к системе новых температурных сенсоров происходит через вызов специалиста. Самостоятельно подключить свой датчик к системе пользователь не может.
- Система поддерживает получение данных от устройств при помощи синхронных вызовов от сервера к датчику
- Система поддерживает получение данных пользователем о температуре в помещении

### 2. Анализ архитектуры монолитного приложения

- Язык программирования: Java
- База данных: PostgreSQL 
- Архитектура: Монолитная, все компоненты системы (обработка запросов, бизнес-логика, работа с данными) находятся в рамках одного приложения.
- Взаимодействие: Синхронное, запросы обрабатываются последовательно.
- Масштабируемость: Ограничена, так как монолит сложно масштабировать по частям.
- Развёртывание: Требует остановки всего приложения.


### 3. Определение доменов и границы контекстов

- Домен: Управление устройствами
  - Поддомен Управление системой отопления
  - Поддомен Получение температуры в помещении


### **4. Проблемы монолитного решения**

- Масштабируемость приложения ограничена
- Развертывание приложения требует остановки, что негативно влияет на доступность системы
- Внесение новых изменений связано с рисками, из-за взаимозависимости кодовой базы в монолите 
- Увеличено время на разработку и тестирование при внедрении новых изменений
